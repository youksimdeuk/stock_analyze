# 챕터 05 — 전문 에이전트 역할 분담

## 원칙
복잡한 작업은 하나의 AI에게 전부 맡기지 않는다. 역할별로 분리하여 업무를 맡기고 상세한 보고서를 요구한다.

---

## 에이전트 역할 정의

### 1. 기획 에이전트 (Planner)
- **역할**: 작업 분석, 설계도/시방서/공정표 초안 작성
- **지시 예시**: "이 기능을 구현하기 위한 설계도 초안을 작성하라. 요구사항, 구현 방식, 예상 결과물을 포함할 것."
- **보고서 형식**: 설계도 + 시방서 문서

### 2. 개발 에이전트 (Developer)
- **역할**: 실제 코드 구현
- **지시 예시**: "공정표 {n}번 항목을 구현하라. 완료 후 무엇을 수정했는지 목록으로 보고하라."
- **보고서 형식**: 수정한 파일 목록 + 변경 사항 요약

### 3. 품질 관리 에이전트 (QA)
- **역할**: 코드 리뷰, 버그 탐지, 보안 점검
- **지시 예시**: "방금 수정된 코드를 검토하라. 누락된 오류 처리, 보안 취약점, 데이터 중복 호출, 일관성 문제를 찾아 보고하라."
- **보고서 형식**: 발견한 문제 목록 + 수정 권고사항

### 4. 테스트 에이전트 (Tester)
- **역할**: 기능 검증, 엣지케이스 테스트
- **지시 예시**: "이 함수의 정상 케이스, 빈값 케이스, API 오류 케이스를 테스트하고 결과를 보고하라."
- **보고서 형식**: 테스트 항목 + 통과/실패 결과

---

## 보고서 필수 포함 항목
단순히 "완료했습니다"로 끝내지 않는다. 반드시 아래를 포함한다:
1. **무엇을 발견했는가** — 문제점, 이슈, 특이사항
2. **무엇을 수정했는가** — 변경된 파일, 함수, 로직
3. **왜 그렇게 판단했는가** — 결정 근거, 트레이드오프

---

## 크로스/셀프 리뷰 프로세스
```
개발 에이전트가 코드 작성
        ↓
품질 관리 에이전트가 동일 코드 재검토
(또는 개발 에이전트 자신이 "다시 읽고 문제점을 찾아라" 지시)
        ↓
발견된 문제 → 수정 → 재검토
```

## Task 도구 활용
Claude Code의 `Task` 도구를 사용해 에이전트를 호출할 때:
- `subagent_type: "general-purpose"` — 기획/분석 작업
- `subagent_type: "Explore"` — 코드베이스 탐색
- `subagent_type: "Bash"` — 명령 실행
- 각 에이전트에게 **구체적인 보고서 형식**을 지시에 포함시킨다.
