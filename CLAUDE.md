# 기업분석 프로젝트 코딩 규칙

## 1. 데이터 호출 최소화
- API/DB/시트 데이터는 가능하면 한 번에 일괄 호출하고 변수에 저장 후 재사용한다.
- 같은 데이터를 두 번 이상 호출하는 코드를 짜지 않는다.
- 반복 호출이 불가피한 경우(연도별 루프, 분기별 루프 등)에만 허용하고, 그 안에서도 공통 파라미터(fs_div, sj_div 등)는 루프 밖에서 1회만 탐색해 재사용한다.

## 2. 잠재적 문제 선제 보고
- 코드를 수정하거나 새로 짤 때, 조금이라도 문제가 예상되면 사용자가 묻기 전에 먼저 말한다.
- 예: 데이터 누락 가능성, API 응답 형식 가정, 엣지케이스, 과거/미래 연도 처리 차이 등.
- 단순 버그 수정이라도 사이드이펙트가 보이면 같이 언급한다.
