# 챕터 04 — 자동 품질 검사 기준

## 원칙
작업 완료 후 반드시 셀프 체크를 수행한다. 억지로 막는 것이 아니라 한 번 더 짚어주면 AI가 스스로 부족한 점을 깨닫고 보완한다.

---

## 완료 후 필수 체크 항목

### 코드 품질
- [ ] 오류 처리(try/except)를 추가했는가?
- [ ] API 응답이 None이거나 빈값일 때 처리했는가?
- [ ] 예외 발생 시 사용자에게 의미 있는 메시지를 출력하는가?

### 데이터 호출
- [ ] 같은 데이터를 두 번 이상 호출하지 않는가?
- [ ] 루프 안에서 불필요한 반복 호출이 없는가?

### 보안
- [ ] API 키가 코드에 하드코딩되어 있지 않은가?
- [ ] 사용자 입력을 그대로 쿼리/명령에 사용하지 않는가?

### 사이드이펙트
- [ ] 수정한 함수의 모든 호출부를 확인했는가?
- [ ] Google Sheets 기존 데이터를 실수로 덮어쓸 위험이 없는가?
- [ ] 반환값 형식 변경이 다른 코드에 영향을 주지 않는가?

### 선제 보고
- [ ] 잠재적 문제를 사용자에게 미리 보고했는가?
- [ ] 엣지케이스를 명시했는가?

---

## 수정 파일 기록 (CCTV)
- 수정된 파일은 `.claude/modification_log.txt`에 자동 기록된다.
- 대규모 수정 후에는 로그를 확인하여 의도치 않은 파일 변경이 없는지 검토한다.
