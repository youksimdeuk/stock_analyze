# 기업분석 프로젝트 — AI 운영 규칙 (목차)

> 상세 내용은 아래 챕터 파일을 필요할 때만 읽으세요.

---

## 핵심 규칙 (항상 적용)

1. API/DB/시트 데이터는 **한 번에 일괄 호출** 후 변수에 저장해 재사용한다.
2. 코드 수정 시 잠재적 문제(누락, 엣지케이스, 사이드이펙트)는 **사용자가 묻기 전에 먼저 말한다.**
3. 큰 작업 시작 전 반드시 **설계도 → 시방서 → 공정표** 3종 문서를 작성하고 승인받는다.
4. 작업은 **한두 개씩 순차적으로** 진행하며, 완료 시마다 공정표를 업데이트한다.
5. 완료 후 **셀프 체크리스트**를 반드시 검토한다.
