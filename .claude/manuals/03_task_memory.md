# 챕터 03 — 작업 기억 시스템

## 원칙
큰 작업은 반드시 **3종 문서(설계도 + 시방서 + 공정표)** 를 작성한 뒤 사용자 승인을 받고 시작한다.
AI가 마음대로 작업을 시작하지 않도록, 승인 전에는 즉시 멈춘다.

---

## 3종 문서 설명

### 1. 설계도 (Plan) — 무엇을 만들 것인가
- 파일명: `.claude/tasks/{작업명}_plan.md`
- 내용: 목표, 요구사항, 구현 방식, 예상 결과물
- 템플릿: `.claude/templates/plan_template.md`

### 2. 시방서 (Context) — 왜 이렇게 결정했는가
- 파일명: `.claude/tasks/{작업명}_context.md`
- 내용: 결정 배경, 참고 자료, 제약 조건, 주의사항
- 템플릿: `.claude/templates/context_template.md`

### 3. 공정표 (Todo) — 어디까지 했는가
- 파일명: `.claude/tasks/{작업명}_todo.md`
- 내용: 단계별 할 일, 완료 여부, 현재 진행 단계
- 템플릿: `.claude/templates/todo_template.md`

---

## 프로세스

```
1. AI가 설계도 초안 작성
        ↓
2. 사용자가 검토 및 수정
        ↓
3. 승인 후 시방서 + 공정표 생성
        ↓
4. 새 대화창에서 공정표 참조하며 작업 시작
        ↓
5. 한두 단계씩 완료 → 공정표 업데이트 → 반복
```

## 순차적 작업 지시 원칙
- 한 번에 모든 것을 시키지 않는다.
- 공정표의 다음 항목 1~2개만 요청한다.
- 완료 후 공정표를 업데이트하고 다음 단계로 넘어간다.
- 대화가 길어지면 새 대화창을 열고 공정표를 참조해 이어간다.

## 체크리스트
- [ ] 큰 작업 시작 전 3종 문서를 작성했는가?
- [ ] 사용자 승인을 받았는가?
- [ ] 공정표를 최신 상태로 유지하고 있는가?
- [ ] 한 번에 너무 많은 작업을 진행하지 않는가?
